1. Stochastic Bandits

  在这一章，我总结了一些简单的强盗算法及其后悔界限。我们首先考虑具有 IID 奖励的基本模型，称为随机强盗（Stochastic Bandits）。

1.1 简单 Bandits 算法

  对于已知的 K 和 T，算法有 K 种可能的行动可供选择，并且进行 T 轮行动选择。记

  • μ(a):=E[Da]\mu (a):=E[\mathcal{D}_a ]μ(a):=E[Da​]，臂 a 的平均奖励，D\mathcal{D}D 是奖励的分布
  • $\mu^\star=max_{a \in \cal A} \mu(a) $，最大平均奖励（对应最优臂）
  • δ(a):=μ⋆(a)−μ(a)\delta(a):=\mu^\star(a)-\mu(a)δ(a):=μ⋆(a)−μ(a)，描述了臂 a 与最优臂 μ⋆\mu^\starμ⋆ 的差距

R(T)=μ∗T−Σt=1Tμ(at)(1)R(T)=\mu^{*} T-\Sigma_{t=1}^{T} \mu\left(a_{t}\right) \tag{1} R(T)=μ∗T−Σt=1T​μ(at​)(1)

于是就可以总结一些简单算法和分析其遗憾。

Algorithm 1: 探索优先（Explore-first） algorithm
1: 探索阶段：每个臂尝试 NNN 次
2: 选择臂： 选择平均奖励最高的臂 a^\hat{a}a^ (最佳臂不止一个时在其中随机选择)
3: 利用阶段: 在剩余的轮次中只玩臂 a^\hat{a}a^

算法思想：

  • 探索阶段的目的是收集关于每个臂的奖励信息，以便有效评估哪个臂表现最好；
  • 选择臂的步骤确保我们能够找到当前最佳臂，以便在后续的利用阶段集中资源；
  • 利用阶段则最大化收益，因为此时我们已确定具有最高平均奖励的臂。

2:   投一枚硬币，成功概率为 ϵt\epsilon_{t}ϵt​

4:    探索：随机选择一个臂
5:   否则:
6:    利用：选择当前平均奖励最高的臂
7: 结束循环

算法思想：这种方法的优势在于能够有效地平衡探索与利用，适应动态环境，同时相对简单易实现。

  • 在每一轮中，算法会以 ϵt\epsilon_{t}ϵt​ 的概率选择探索（随机选择一个臂），以 1−ϵt1 - \epsilon_{t}1−ϵt​ 的概率选择利用（选择当前奖励平均值最高的臂）。
  • 这种做法允许算法在保持一定的探索行为的同时，主要依赖于当前已知的最佳臂进行决策。
  • ϵ\epsilonϵ 的值可以随着时间变化，通常会初始设定为较大值以鼓励探索，而随着时间的推移，逐渐减小以增加利用。

1.2 自适应探索算法

定义 2（置信区间）：对于每个臂 a 和 第 ttt 轮,

\begin{align*} r_{t}(a) & =\sqrt{2 \log (T) / n_{t}(a)} \quad \text { (置信半径，confidence radius) } \\ U C B_{t}(a) & =\bar{\mu}_{t}(a)+r_{t}(a) \quad \text { (上置信界，upper confidence bound) } \tag{2}\\ L C B_{t}(a) & =\bar{\mu}_{t}(a)-r_{t}(a) \quad \text { (下置信界，lower confidence bound) } \end{align*}

用集中不等式我们就有置信区间 [LCBt(a),UCBt(a)]\left[L C B_{t}(a), U C B_{t}(a)\right][LCBt​(a),UCBt​(a)] 和置信半径 rt(a)r_{t}(a)rt​(a)。于是我们就有一些自适应探索算法：

Algorithm 3: 针对两个臂的连续算法

  1. 在两个臂之间交替进行，直到在某个偶数回合 $ t $ 时满足条件 $ UCB_{t}(a) < LCB_{t}(a’) $。
  2. 放弃臂 $ a $，并从此以后只使用臂 $ a’ $。

算法思想：该算法专门用于有两个臂的情况

  • 在交替选择这两个臂的过程中，算法会根据上置信界（UCB）和下置信界（LCB）的比较来决定何时放弃表现较差的臂。
  • 一旦满足条件 $ UCB_{t}(a) < LCB_{t}(a’) $，则可以认为臂 $ a’ $ 拥有更优的表现，算法将停止对臂 $ a $ 的使用。
  • 这种方法有效地将探索和利用结合在一起，只针对两个臂进行比较，使决策过程较为简单且高效。

定理 3（见书 P9 ）：连续算法对所有 t≤Tt \leq Tt≤T 满足

\begin{equation*} \mathbb{E}[R(t)]=O(\sqrt{t \log T}) \tag{3} \end{equation*}

这是两个臂的情形，对于多臂的情况：

Algorithm 4: 连续淘汰算法

  1. 所有臂最初被设为活跃状态。
  2. 循环：
  3.   每个活跃臂各玩一次。
  4.   根据以下规则禁用所有满足条件的臂 aaa：设当前轮次为 ttt，如果 UCBt(a)<LCBt(a′)UCB_{t}(a) < LCB_{t}(a')UCBt​(a)<LCBt​(a′) 对于某个其他臂 a′a'a′（禁用规则）。
  5. 结束循环

算法思想：

  • 在这个算法中，所有臂最开始都是活跃的，每轮每个活跃的臂都尝试一次。
  • 然后使用上置信界（UCB）和下置信界（LCB）来确定哪些臂应该被禁用。具体来说，只要某个臂的上置信界小于另一个臂的下置信界，就可以将该臂禁用。这意味着我们可以确保不会考虑那些在统计上表现不好或潜在表现不佳的臂。
  • 这个过程会持续进行，直到只剩下一个臂或者达到设定的终止条件。

这种算法的优点是能够高效地排除明显不佳的选择，集中资源在更有潜力的臂上，特别适用于对于臂的性能未知而需要动态调整的情况。

定理 4（证明见书 P11 ）：连续淘汰算法对所有 t≤Tt \leq Tt≤T 满足遗憾

\begin{equation*} \mathbb{E}[R(t)]=O(\sqrt{K t \log T}) \tag{3} \end{equation*}

定理5： 连续淘汰算法满足遗憾界

\begin{equation*} \mathbb{E}[R(t)] \leq O(\log T)\left[\sum_{\text {arms a with } \mu(a)<\mu^{*}(a)} \frac{1}{\mu^{*}(a)-\mu(a)}\right] \tag{4} \end{equation*}

注意，我们只能使用 UCBtUCB_{t}UCBt​ 来确定哪条臂更好，因为一条臂 aaa 的上置信界 UCBt(a)U C B_{t}(a)UCBt​(a) 可能很大的原因有两个（或两个原因的组合）：

  • 均值 mˉut(a)\bar{m} u_{t}(a)mˉut​(a) 很大
  • 置信半径 rt(a)r_{t}(a)rt​(a) 很大，这种情况下是因为这条臂还没有被探索很多次

所以我们有了这个算法：

Algorithm 5 ： UCB1 算法

  1. 每个臂尝试一次
  2. 对于 每轮 $ t = 1, \ldots, T $ 执行：
  3.   选择臂 $ a $，使得 $ UCB_{t}(a) $ 最大化。
  4. 结束循环

算法思想：

  • 在此算法中，初始步骤是对每个臂进行一次试验，以获取初步的奖励信息。在每个后续轮次中，计算每个臂的上置信界（UCB），选择 UCB 最大的臂，然后进行利用并更新奖励。
  • 通过这种方式，算法能够在逐步了解臂的性能时，优先选择收益最有保障的臂。

3. Bayesian Bandits

  考虑一种简单的贝叶斯 bandit 算法，称为汤普森采样（Thompson Sampling）。对于每一轮 ttt 和臂 aaa，算法计算出 aaa 是最佳手臂的后验概率，并以此概率对 aaa 进行采样。

Algorithm 6 汤普森采样算法

1： 对于每轮 $ t=1,2,\ldots $ 执行:
2：  观察历史信息 $ H_{t-1}=H $，这是一个可行的 (t-1)-历史 H
3：  从分布 $ p_{t}(\cdot \mid H) $ 中独立抽取臂 $ a_{t} $，这里的分布定义为：

pt(a∣H):=P[a∗=a∣Ht−1=H]p_{t}(a \mid H) := \mathbb{P}\left[a^{*}=a \mid H_{t-1}=H\right] pt​(a∣H):=P[a∗=a∣Ht−1​=H]

其中 a∗a^{*}a∗ 表示最优臂， $ H $ 是之前观察到的历史信息
4： 结束循环

算法思想：

  • 在每一轮中，算法首先观察最近一次的历史数据 $ H_{t-1} $。这个历史数据包含了之前的选择和结果，帮助算法基于已有的信息进行决策。
  • 根据历史信息 $ H $，算法为每个臂计算一个概率分布 $ p_{t}(a \mid H) $。这个分布反映了在给定历史下，某个臂 $ a $ 是最优臂的概率。随后，算法从这个分布中独立抽取一个臂 $ a_{t} $，该臂将被选择执行。

汤普森采样通过利用历史数据的概率模型，允许算法根据当前信息选择臂。由于其概率抽样的特点，汤普森采样在实践中表现优异，它能够有效应对动态和不确定的环境，帮助实现收益的最大化。与其他基于确定性策略的方法相比，汤普森采样能更好地平衡探索新选项与利用已知选项之间的矛盾，因此在许多实际应用中非常受欢迎，如在线广告、推荐系统等领域。

Algorithm 7 汤普森采样的等价版本

1: 对于每轮 $ t=1,2,\ldots $执行:
2： 观察历史信息 Ht−1=HH_{t-1}=HHt−1​=H ，这是一个可行的 (t-1)-历史 H
3： 从后验分布 PH\mathbb{P}_{H}PH​ 中采样均值奖励向量 μt\mu_tμt​
4： 根据 μt\mu_tμt​ 选择最佳臂 a~t\tilde{a}_ta~t​
5：结束循环

算法思想：汤普森采样的等价版本通过不同的方式实现决策过程

  • 在每一轮，算法观察之前的历史数据 $ H_{t-1} $，这为后续的决策提供了必要的信息。

  • 从后验分布 PH\mathbb{P}_HPH​ 中采样均值奖励向量 μt\mu_tμt​。这个步骤利用历史数据更新各个臂奖励的统计特性，使得算法能够获取更新的奖励估计。

  • 根据采样得到的均值奖励向量 $ \mu_{t} $，算法选择当前看来最有可能提供最佳奖励的臂 $ \tilde{a}_{t} $。这样，臂的选择是基于对期望奖励的估计来进行的。

4. Lipschitz bandits

  在特殊情况下，手臂对应于区间 X=[0,1]X=[0,1]X=[0,1] 中的点，之前臂的数量是离散的，此时可以从一个区间的实数中取值作为 “臂”，每一个取值都对应一个奖励分布。那么对任意两个臂 x,y∈Xx, y \in Xx,y∈X 期望奖励服从 Lipschitz 条件：

\begin{equation*} |\mu(x)-\mu(y)| \leq L|x-y| \tag{13} \end{equation*}

我们可以看到，这个 Lipschitz 条件描述了 “相似 ”的手臂具有相似的预期回报。当这个 bandit 模型有无穷多条手臂时，一个简单的解决方案就是固定离散化：用有限的手臂 S∈X\mathcal{S} \in XS∈X 来近似整个区间 XXX，然后只关注 S\mathcal{S}S 中的臂，然后运行一个现成的算法 ALG ( 如UCB1 或连续淘汰算法) 而这样的 Lipschitz 条件可以保证这些有限的手臂拥有足够的信息。

注意，在 S\mathcal{S}S 上增加更多的点使其更接近 XXX，但同时也增加了 ALG 的遗憾，因此需要权衡离散化策略：固定离散化、均匀离散化、自适应离散化。

定理 10 ：考虑具有 Lipschitz 常数 LLL 和时间跨度 TTT 的连续臂 bandit 算法。使用满足 Lipschitz 条件和（离散化）步长为 ϵ=(TL2/log⁡T)−1/3\epsilon=\left(T L^{2} /\log T\right)^{-1 / 3}ϵ=(TL2/logT)−1/3 的 ALG 算法进行均匀离散化，可以得到遗憾界

\begin{equation*} \mathbb{E}[R(T)] \leq L^{1 / 3} T^{2 / 3}\left(1+c_{A L G}\right)(\log T)^{1 / 3} \tag{14} \end{equation*}

其中 cALGc_{A L G}cALG​ 是算法特有的绝对常数。这里的主要是 O~(L1/3T2/3)\tilde{O}\left(L^{1 / 3} T^{2} / 3\right)O~(L1/3T2/3) 起作用，所以常数和对数依赖性并不那么重要。实际上，在最坏的情况下，均匀离散化是最优的：我们有一个 Ω(L1/3T2/3)\Omega\left(L^{1 / 3} T^{2 / 3}\right)Ω(L1/3T2/3) 的遗憾下限。

定理 11：假设 ALG 是时间跨度为 TTT 和 Lipschitz 常数为 LLL 的连续臂 bandit 算法。存在一个问题实例 L=L(x∗,ϵ)L=L\left(x^{*}, \epsilon\right)L=L(x∗,ϵ)，对于某个 x∈[0,1]x \in[0,1]x∈[0,1] 和 ϵ>0\epsilon>0ϵ>0，使得

\begin{equation*} \mathbb{E}[R(T) \mid L] \geq\left(L^{1 / 3} T^{2 / 3}\right) \tag{15} \end{equation*}

更一般的情况，对任意臂 x,yx,yx,y ，Lipschitz 条件可描述为：

\begin{equation*} |\mu(x)-\mu(y)| \leq \mathcal{D}(x, y) \tag{16} \end{equation*}

其中，D\mathcal{D}D 是一种度量。

定义3： 一个子集 $ S \subseteq X $ 被称为 ϵ\epsilonϵ-mash（ϵ>0\epsilon > 0ϵ>0），如果对于任意点 $ x \in X $，存在某个点 $ y \in S $，使得 $ D(x, y) \leq \epsilon $，即每个点 $ x $ 与 $ S $ 的距离都在 ϵ\epsilonϵ 之内。

那么我们就有了这种更一般情况下的遗憾约束：
定理 12: 考虑时间跨度为 T 的 Lipschitz bandits。在选择 ϵ\epsilonϵ-mesh 时，使用均匀离散化的方法和算法 ALG 能够达到的遗憾界为：

\begin{equation*} \mathbb{E}[R(T)] \leq \inf _{\epsilon>0, \epsilon-\text { mesh } S} \epsilon T+c_{\mathrm{ALG}} \sqrt{|S| T \log T} \tag{17} \end{equation*}

在 $ d $ 维欧氏空间中使用 $ L_{p} $ 度量时，我们可以得到一个显式的形式。在更一般的情况下，可以通过“覆盖维数”的定义来获得这种形式。我们省略了这些不重要的细节，以便快速进入我们的主要算法。

  • 具体的度量形式取决于 p 的值，常见的如 L1（曼哈顿距离）和 L2（欧氏距离）。
  • “覆盖维度”是一个拓扑和几何中的概念，涉及到用小的球体或其他形状来覆盖一个集合的性质。意味着可以在更一般的情况下理解空间中对象如何相互“接近”。

现在我们可以描述算法。它包含两个部分：激活步骤和选择步骤。我们考虑定义为 $ B_{t}(x) = {y \in X: \mathcal{D}(x, y) \leq r_{t}(x)} $ 的置信球，球外的臂是不活跃的：

  • 激活步骤：我们寻找一个不在所有活跃臂的置信球中的臂。直观上我们可以看到，如果某个臂非常接近某个活跃臂，由于 Lipschitz 条件，我们就不需要对其进行探索。
  • 选择步骤：我们选择一个处于活跃状态的臂。这个想法就像 UCB1 算法。如果某个臂 $ x $ 在时间 $ t $ 是活跃的，我们定义

\begin{equation*} \operatorname{index}_{t}(x)=\bar{\mu}_{t}(x)+2 r_{t}(x) \tag{18} \end{equation*}

选择规则非常简单：选择 index 最大的活跃臂。

Algorithm 8 自适应离散化的缩放算法（Zooming algorithm）
1： 初始化：建立一个活跃臂的集合 $ S $，初始时为空，即 $ S \leftarrow \emptyset $。
2： 对于 每轮 $ t=1,2,\ldots $执行:
3：  如果 某个臂 $ y $ 未被覆盖在活动臂的置信球内 那么：
4：   选择任意一个这样的臂 $ y $ 并“激活”它：更新活跃臂集合 $ S \leftarrow S \cup {y} $。
5：  结束条件
6：  玩一个具有最大 $ \operatorname{index}_{t}(x) $ 的活跃臂 $ x $。
7：结束循环

算法思想：该算法通过逐步激活新臂并根据每个臂的“指数”进行选择，从而平衡探索和利用。

  • 初始化活动臂集合 $ S $，开始时没有激活的臂。

  • 条件检查：在每一轮中，算法会检查是否存在未被覆盖在当前活动臂置信球（即通过已有臂计算出的奖励估计范围内）的臂 $ y $。置信球用于定量表示臂的潜在性能范围，以便在选择臂时做出更有把握的选择。

  • 激活新臂：如果找到符合条件的臂 $ y $，则将其加入活动臂集合 $ S $。这样，算法能够逐步引入具有探索潜力的新臂，而不是在一开始就一次性激活所有可能的臂。

  • 选择活跃臂：在活动臂中，算法选择具有最大 $ \operatorname{index}_{t}(x) $ 的臂进行游戏。

为了处理具有上下文 ∣X∣|\mathcal{X}|∣X∣ 较大的上下文赌博机，我们可以采取以下方法之一：

  1. 假设某种结构：我们可以假设上下文满足某些结构条件，例如 Lipschitz 条件。这种假设使得我们能够在面对高维上下文时，更有效地进行算法设计和优化。

  2. 采用线性上下文赌博机的设置：在这种情况下，我们假设奖励与上下文之间存在某种线性关系，从而简化模型和算法的复杂性。这种方法可以利用线性回归和相关技术进行有效学习。

  3. 改变目标：另一种方法是改变算法的目标，例如通过聚焦于优化不同于传统最大化奖励的目标，从而使算法在处理大型上下文时更加灵活和适应。

以上方法都有助于在面对复杂上下文时提高上下文老虎机的有效性和可操作性。
